# Flutter 技术选型分析

> **阅读时机**: 评估技术方案、做架构决策时

---

## 为什么选择 Flutter

### 与 Electron 对比

| 指标 | Electron | Flutter | 选择原因 |
|------|----------|---------|----------|
| 内存占用 | 300MB+ | 100-150MB | ✅ 性能优先 |
| 安装包大小 | 80-150MB | 20-40MB | ✅ 用户体验 |
| 启动速度 | 2-5s | 0.5-1s | ✅ 性能优先 |
| 跨平台一致性 | 依赖 Chromium | 完全一致 | ✅ UI一致性 |
| 开发效率 | 高 | 高 | 相当 |
| 图可视化生态 | G6 成熟 | **需自研** | ⚠️ 需投入 |

### 选择 Flutter 的理由

1. **性能优势**: 内存减少 60%+，启动速度提升 4-5 倍
2. **现代化 UI**: Material Design 3，开箱即用
3. **单一代码库**: 一套代码支持桌面/移动端/ Web
4. **Dart 语言**: 现代、类型安全、异步支持好

## 核心技术选型

### 状态管理: Riverpod

```yaml
# 推荐: flutter_riverpod
dependencies:
  flutter_riverpod: ^2.4.0
```

**选择理由**:
- 编译时安全，避免 ProviderNotFoundException
- 支持自动销毁，内存管理友好
- 测试友好，无需 Provider 容器
- API 简洁，学习曲线低

**对比**:
| 方案 | 优势 | 劣势 |
|------|------|------|
| Riverpod | 类型安全、测试友好 | 相对较新 |
| Provider | 官方推荐 | 运行时错误风险 |
| Bloc | 模式清晰 | 模板代码多 |
| GetX | 全功能 | 过度封装 |

---

### 本地存储: Hive

```yaml
dependencies:
  hive_flutter: ^1.1.0
```

**选择理由**:
- 纯 Dart 实现，跨平台一致
- 性能优于 SQLite
- 无需原生依赖
- API 简洁

**对比**:
| 方案 | 性能 | 复杂度 | 推荐场景 |
|------|------|--------|----------|
| Hive | 高 | 低 | 键值存储 ✅ |
| Isar | 很高 | 中 | 大数据量 |
| SQLite | 中 | 高 | 复杂查询 |

---

### 表格组件: Syncfusion

```yaml
dependencies:
  syncfusion_flutter_datagrid: ^24.1.0
```

**选择理由**:
- 商业级组件，功能完整
- 支持编辑、排序、筛选
- 性能好，支持大数据量
- 免费版够用

**对比**:
| 方案 | 功能 | 性能 | 推荐度 |
|------|------|------|--------|
| Syncfusion | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ✅ 推荐 |
| PlutoGrid | ⭐⭐⭐⭐ | ⭐⭐⭐ | 备选 |
| DataTable2 | ⭐⭐⭐ | ⭐⭐⭐ | 简单场景 |

---

### 模板引擎: mustache_template

```yaml
dependencies:
  mustache_template: ^2.0.0
```

**选择理由**:
- Dart 原生实现
- 语法简单，易于维护
- 支持条件、循环等逻辑

## 图可视化方案 ⚠️ 重点

### Flutter 生态现状

| 库 | 功能 | 成熟度 | 适用性 |
|---|------|--------|--------|
| flutter_graphview | 基础图布局 | ⭐⭐ | 不够用 |
| graphview | 简单图形 | ⭐⭐ | 不够用 |
| **CustomPainter 自绘** | 完全控制 | ⭐⭐⭐⭐⭐ | ✅ 推荐 |

### 推荐方案: CustomPainter 自研

**原因**:
1. Flutter 无成熟的 ER 图组件
2. 现有图库不支持复杂交互(拖拽编辑、连线编辑)
3. 自研可完全控制功能

**预估投入**: 2-3 周

**关键实现**:
```
ERDiagramWidget
    ├── InteractiveViewer (缩放/平移)
    ├── CustomPaint (画布绘制)
    │       ├── NodePainter (节点)
    │       └── EdgePainter (连线)
    └── GestureDetector (交互处理)
```

详见: [关系图编辑器](../features/relation-graph/README.md)
