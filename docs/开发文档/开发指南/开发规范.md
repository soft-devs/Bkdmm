# 开发规范

> 阅读时机: 团队协作、代码审查

---

## 代码风格

### 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 文件 | 小写下划线 | `project_service.dart` |
| Widget | `*_view.dart` | `home_view.dart` |
| Service | `*_service.dart` | `project_service.dart` |
| Provider | `*_provider.dart` | `project_provider.dart` |
| 类 | 大驼峰 | `ProjectService` |
| 变量 | 小驼峰 | `projectName` |
| 私有变量 | `_`前缀 | `_project` |

---

## 目录规范

```
features/{模块}/
├── models/
├── services/
├── providers/
├── widgets/
└── views/
```

## 状态管理

```dart
// 推荐: StateNotifierProvider
final projectProvider = StateNotifierProvider<ProjectNotifier, ProjectState>((ref) {
  return ProjectNotifier();
});

class ProjectState {
  final Project? project;
  final bool isDirty;
}
```

---

## Commit 规范

```
<type>(<scope>): <subject>

type:
  - feat: 新功能
  - fix: 修复
  - docs: 文档
  - refactor: 重构

示例:
  feat(editor): 添加字段拖拽
  fix(project): 修复保存失败
```
